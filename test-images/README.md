# OCRテスト用画像ディレクトリ

このディレクトリは、レシートOCR機能をテストするための画像を保存する場所です。

## テスト方法

1. このディレクトリにレシート画像を配置します（例: `receipt1.jpg`）
2. テストスクリプトを実行します:

```bash
npx tsx scripts/test-ocr.ts ./test-images/receipt1.jpg
```

## サポートされているファイル形式

### 画像形式（推奨）

- **標準画像**: JPG, JPEG, PNG, GIF, BMP, WEBP, TIFF
- **iOS画像**: HEIC, HEIF（iPhoneで撮影した画像）
- **RAW画像**: RAW, ICO

### ドキュメント形式

- **PDF**: 複数ページ対応（Google Vision APIの制限により一部PDFは処理できない場合があります）

## 推奨される画像品質

- **解像度**: 最低でも1000x1000ピクセル
- **ファイルサイズ**: 10MB以下
- **撮影のポイント**:
  - レシート全体が写っていること
  - 明るい場所で撮影
  - ピントが合っていること
  - 文字が読みやすいこと
  - 影や反射が少ないこと

## テスト用の画像を取得する方法

### 方法1: スマホでレシートを撮影
最も簡単で正確な方法です。実際のレシートをスマホで撮影してこのディレクトリに配置してください。

### 方法2: サンプル画像を使用
以下のようなサイトからテスト用のレシート画像を取得できます:

```bash
# 簡単なテキスト画像（英数字のみ）でテスト
curl -L "https://via.placeholder.com/800x600/FFFFFF/000000?text=RECEIPT%0AABC+Store%0ATotal:+$19.99%0ADate:+2024-01-09" -o test-images/simple-text.png
```

### 方法3: 自分で作成
テキストエディタで簡単なレシート風の画像をスクリーンショットで作成することもできます。

## トラブルシューティング

### エラー: "No text detected in the image"

**原因**:
- 画像が不鮮明
- 文字が小さすぎる
- コントラストが低い

**解決策**:
- より鮮明な画像を使用する
- レシートに近づいて撮影する
- 明るい場所で撮影する
